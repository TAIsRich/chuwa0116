## 3.  What is the Aspect Oriented Programming?
    Aspect-oriented programming (AOP) is a programming technique that supports the separation of cross-cutting concerns in order to increase modularity.

    AOP is typically used to implement cross-cutting concerns, which implies that it defines functionality that is required in multiple places across an application in one place. You can add new functionality before or after a method is executed like transaction management, logging or security which cut across multiple types and objects (often termed crosscutting concerns).

## 4.  What are the advantages and disadvantages of Spring AOP?
    Advantages of Spring AOP
    It is easy to configure.
    Spring AOP is implemented in pure Java, so separate compilation unit or separate class loader are not required.
    It utilizes Spring's IOC container for dependency injection.
    Can create aspects using @AspectJ annotation based or using XML based.
    It integrates cross-cutting concerns into the classes.

    Disadvantages of Spring AOP
    Debugging the AOP framework-based application code is a little challenge.
    Only methods with a public visibility will be recommended, not those with a private, protected, or default visibility.
    Aspects cannot be advised by other aspects. This is because once a class is marked as an aspect (using XML or annotation), Spring prevents it from being auto-proxied.

## 5.  What is Aspect in Spring AOP?
    An aspect is a cross-cutting module that combines advice and pointcuts. A standard class tagged with the @Aspect annotation can be used to implement an aspect.

## 6.  What is Pointcut in Spring AOP?
    A pointcut is an expression that chooses one or more join points at which advice is given. Pointcuts can be defined using expressions or patterns. It supports a number of expressions that correspond to the join points.

## 7. What is the Join point in Spring AOP?
    A join point is a place in the application where an AOP aspect is applied. It could also be a specific advice execution instance. A join point in AOP can be a method execution, exception handling, changing the value of an object variable, and so on.

## 8.  What does it mean by Advice and its types in Spring AOP?
    The advice is an action which we take before or after the method execution. In the Spring AOP framework, there are five types of advice: before, after, after-returning, after-throwing, and around advice. Advice is taken at a specific join point.

## 10. When to use Spring Batch?
    Spring Batch is a framework designed to help developers build batch processing applications. Batch processing is used to process a large volume of data or transactions in a single job run, such as processing payroll, generating reports, or updating customer data.

    Spring Batch is suitable for applications that require the following features:

    Large volume data processing: When an application needs to process a large amount of data, Spring Batch can handle the processing of data in chunks, making it easier to manage memory consumption and improve performance.
    Job management: Spring Batch provides a comprehensive job management system that allows developers to manage batch jobs and schedule them for execution. It provides features such as job restartability, job monitoring, and job execution history.
    Robust exception handling: Batch processing applications must have robust exception handling capabilities to ensure that job processing can continue in the event of errors or failures. Spring Batch provides a range of exception handling strategies, such as retry, skip, and fault-tolerant steps.
    Parallel processing: When processing large volumes of data, parallel processing can help improve performance. Spring Batch provides support for parallel processing and partitioning.
    Transaction management: Spring Batch provides transaction management capabilities to ensure that batch jobs are processed in a transactional manner. This means that if any errors occur during the job, the data can be rolled back to the previous state.

## 11. How Spring Batch works?
    Spring Batch is a framework designed to help developers build batch processing applications. Spring Batch provides a robust job processing framework that supports parallel processing, error handling, and transaction management.

    Here's a high-level overview of how Spring Batch works:

    Job Definition: Developers define a job using Spring Batch's Job Specification Language (JSL) or through the Java-based job configuration API. A job consists of one or more steps, which are the individual units of work that make up the job.
    Step Processing: Each step consists of one or more item processing tasks, which are performed on a set of input data. The input data can be obtained from a variety of sources, such as a database, flat file, or message queue.
    Chunk Processing: Spring Batch processes data in chunks, meaning that data is read in chunks, processed, and then written out in chunks. This approach minimizes memory consumption and improves performance.
    Item Processing: Each item is processed by an item processor, which is responsible for performing any necessary transformations or business logic on the data.
    Job Execution: Once the job is defined, it can be executed by the JobLauncher, which is responsible for starting the job execution. Spring Batch provides a range of job execution strategies, such as sequential, parallel, and remote.
    Job Monitoring and Management: Spring Batch provides a range of job monitoring and management features, such as job status tracking, job execution history, and job restartability. Developers can use these features to monitor and manage batch jobs as they run.

## 13. What is the cron expression?
    A cron expression is a string of six or seven fields that specify a schedule for a job in a Unix-based operating system. A cron expression defines the schedule by specifying the date and time when a job should run, using a combination of numbers and special characters.

    The six fields in a cron expression are:

    Minute: A number between 0 and 59 that specifies the minute when the job should run.
    Hour: A number between 0 and 23 that specifies the hour when the job should run.
    Day of the month: A number between 1 and 31 that specifies the day of the month when the job should run.
    Month: A number between 1 and 12 or a three-letter abbreviation that specifies the month when the job should run.
    Day of the week: A number between 0 and 7 or a three-letter abbreviation that specifies the day of the week when the job should run (0 or 7 represents Sunday).
    Year (optional): A number that specifies the year when the job should run.

    Task scheduling: Spring Task provides a TaskScheduler interface that can be used to schedule tasks to run periodically or at specific times. The interface allows developers to configure the task scheduling using a variety of parameters, such as a fixed delay, a fixed rate, or a cron expression.
    Task execution: Spring Task provides a simple interface for defining the tasks that need to be executed. Developers can implement the Runnable interface to define the code that needs to be executed as part of the task.
    Task management: Spring Task provides several features for managing scheduled tasks, such as the ability to cancel tasks, change the scheduling parameters, and track the execution history of tasks.

## 16. What is Filter and any filter example?
    n the context of web development, a filter is a component that intercepts and processes HTTP requests and responses between a client and a server. Filters are used to modify or transform the request or response before it is processed by the server or sent back to the client.

    Filters are implemented in the Java Servlet API and are used to implement cross-cutting concerns, such as security, logging, and caching. They can also be used to preprocess requests or postprocess responses, such as compressing data or adding headers.

## 17.  What is the Interceptor? What we can do with interceptor?
    In the context of web development, an interceptor is a component that intercepts and processes HTTP requests and responses in a similar way to filters. However, interceptors are implemented in the Spring Framework and are used to implement cross-cutting concerns such as logging, security, and transaction management.

    Interceptors are implemented as beans in the Spring Framework and can be used to modify the request or response before it is processed by the controller or view, or after it has been processed.


## 18. In Interceptor, What is preHandle? What is postHandle?
    In the Spring Framework's HandlerInterceptor interface, preHandle and postHandle are two methods that intercept and process HTTP requests and responses in a Spring MVC application.

    preHandle: The preHandle method is called before the request is handled by the controller. It allows interceptors to execute logic before the controller processes the request. This method returns a boolean value indicating whether the request should be further processed by the handler or not. If the return value is false, the request will be aborted and no further processing will occur. Otherwise, the request will be passed to the controller for processing.
    postHandle: The postHandle method is called after the controller has processed the request but before the response is sent to the client. It allows interceptors to execute logic after the controller has completed its processing. This method can modify the model and view used to generate the response, or it can add additional attributes to the model.

## 19. What is Swagger?
    Swagger is an open-source framework that allows developers to design, build, document, and test RESTful APIs. It provides a range of tools and features for building and documenting APIs, including a user interface for exploring and testing APIs, automated documentation generation, and client code generation.

    The Swagger framework includes several components:

    Swagger Editor: A browser-based editor for designing and editing OpenAPI Specification (formerly known as Swagger Specification) files, which describe the API's structure and behavior.
    Swagger UI: A user interface for visualizing and exploring APIs based on their OpenAPI Specification. Swagger UI allows developers to test and interact with APIs directly from the browser.
    Swagger Codegen: A code generation tool that can generate client libraries for a variety of programming languages based on the API's OpenAPI Specification.
    Swagger Core: A set of Java libraries for building RESTful APIs that support the OpenAPI Specification.
